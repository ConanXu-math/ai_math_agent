# 平台接口列表与调用顺序

本文档定义**平台层对外暴露的 API**（不是直接暴露 OpenCode / OpenEvolve / Numina 的原始接口），以及一次完整运行的**建议调用顺序**。

---

## 一、平台要实现的接口列表

### A. 任务与运行

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/v1/tasks` | 创建研究任务（目标、约束、预算、策略等） |
| GET  | `/v1/tasks/{task_id}` | 任务详情与当前阶段 |
| POST | `/v1/tasks/{task_id}/runs` | 启动一次 run（可指定从哪阶段开始） |
| GET  | `/v1/runs/{run_id}` | run 状态、阶段进度、失败原因 |
| POST | `/v1/runs/{run_id}/cancel` | 取消运行 |
| POST | `/v1/runs/{run_id}/resume` | 从失败点续跑 |

### B. 阶段执行（可选暴露，便于调试与手动重试）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/v1/runs/{run_id}/stages/draft` | 生成初始方案/代码草案（走 OpenCode） |
| POST | `/v1/runs/{run_id}/stages/evolve` | 触发进化（走 OpenEvolve） |
| POST | `/v1/runs/{run_id}/stages/formalize` | 代码转数学形式并做 R1–R7 校验（走 Numina） |
| POST | `/v1/runs/{run_id}/stages/lean` | 数学形式转 Lean 并修复（Numina agent/API） |
| POST | `/v1/runs/{run_id}/stages/report` | 汇总结论与证据链 |

### C. 产物与审计

| 方法 | 路径 | 说明 |
|------|------|------|
| GET  | `/v1/runs/{run_id}/artifacts` | 列出本 run 的产物索引（代码、日志、数学式、Lean 等） |
| GET  | `/v1/artifacts/{artifact_id}` | 读取产物内容或下载地址 |
| GET  | `/v1/runs/{run_id}/timeline` | 按时间返回每个动作与外部调用记录 |
| GET  | `/v1/runs/{run_id}/metrics` | 分数、通过率、成本、耗时等 |

---

## 二、建议的调用顺序（一次完整 run）

1. **创建任务**  
   `POST /v1/tasks`，得到 `task_id`。

2. **创建并启动 run**  
   `POST /v1/tasks/{task_id}/runs`，得到 `run_id`。

3. **草案阶段**  
   通过 OpenCodeAdapter：创建 project/session，发送「根据问题生成初始程序」的 prompt，收集生成代码，写入 `draft/` 产物。

4. **进化阶段**  
   通过 OpenEvolveAdapter：以 draft 代码为初始程序执行进化；收集 `best_program.py` 与评估指标，写入 `evolve/` 产物。

5. **形式化阶段**  
   通过 NuminaAdapter：对 best 程序做 code→math→R1–R7；若未通过，写回失败原因并终止或进入重试策略。

6. **Lean 阶段**  
   通过 NuminaAdapter：math→Lean（优先 `TRANSLATE_BACKEND=numina_agent`，失败回退 `api`）；收集 Lean 文件与诊断，写入 `lean/` 产物。

7. **报告阶段**  
   汇总代码、数学形式、Lean、评估与诊断，生成报告与证据链，写入 `report/`；将 run 状态置为 `succeeded` 或 `failed`，并写入 timeline 与 metrics。

8. **查询与审计**  
   通过 `GET /v1/runs/{run_id}`、`/artifacts`、`/timeline`、`/metrics` 做复现与对比。

---

## 三、与底层组件的对应关系

| 平台阶段   | 底层组件        | 主要输入           | 主要输出                 |
|------------|-----------------|--------------------|--------------------------|
| draft      | OpenCodeAdapter | 问题描述、约束     | 初始代码、prompt 记录    |
| evolve     | OpenEvolveAdapter| 初始代码、config   | best_program、分数、logs |
| formalize  | NuminaAdapter    | best 代码          | math_form、R1–R7 结果    |
| lean       | NuminaAdapter    | math_form          | .lean 文件、诊断         |
| report     | 平台汇总         | 各阶段产物         | 报告、证据链、metrics    |

平台层只依赖上述接口与调用顺序；内部实现可随 OpenCode / OpenEvolve / Numina 的版本迭代而替换适配器，而不影响对外 API 契约。
